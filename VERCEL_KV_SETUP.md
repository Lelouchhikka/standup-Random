# Настройка Vercel KV для хранения данных

## Что такое Vercel KV?

Vercel KV - это встроенное хранилище данных на базе Redis, интегрированное прямо в Vercel. Идеально подходит для вашего приложения!

## Преимущества:

- ✅ **30,000 команд/месяц** на бесплатном плане (обновляются каждый месяц!)
- ✅ Интеграция в один клик - никаких API ключей
- ✅ Очень быстрый (Redis)
- ✅ Автоматическая настройка переменных окружения
- ✅ Работает в обычном режиме и инкогнито

## Настройка (3 минуты):

### Шаг 1: Откройте ваш проект в Vercel

1. Перейдите на https://vercel.com/dashboard
2. Выберите проект **standup-Random**

### Шаг 2: Включите Vercel KV

1. В верхнем меню нажмите на вкладку **Storage**
2. Нажмите **Create Database**
3. Выберите **KV (Redis)**
4. Введите имя для базы: `standup-participants-db` (или любое другое)
5. Выберите регион ближайший к вам (например: **Washington, D.C., USA (East)** или **Frankfurt, Germany**)
6. Нажмите **Create**

### Шаг 3: Подключите к проекту

1. После создания базы нажмите **Connect to Project**
2. Выберите ваш проект **standup-Random**
3. Выберите окружения: **Production**, **Preview**, **Development**
4. Нажмите **Connect**

**Готово!** ✅ Vercel автоматически добавил все необходимые переменные окружения.

### Шаг 4: Передеплойте проект

Чтобы изменения вступили в силу:

1. Перейдите на вкладку **Deployments**
2. Нажмите на три точки (•••) у последнего деплоя
3. Выберите **Redeploy**
4. Подтвердите действие

## Проверка работы

После деплоя:

1. Откройте ваше приложение в **обычном** окне браузера
2. Откройте то же приложение в **режиме инкогнито**
3. Добавьте участника в одном окне
4. Обновите страницу в другом окне

Участник должен появиться в обоих окнах! 🎉

## Мониторинг использования

Чтобы посмотреть сколько запросов используете:

1. Проект → **Storage** → Ваша KV база
2. Вкладка **Usage** показывает количество команд за месяц

Лимит: **30,000 команд/месяц** (обновляется каждый месяц)

## Технические детали

Приложение использует всего 2 команды:
- `GET standup-participants` - загрузка списка
- `SET standup-participants` - сохранение списка

При нормальном использовании вы будете далеко от лимита!
