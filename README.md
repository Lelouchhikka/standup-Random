# Stand-up Randomizer

Современное веб-приложение для генерации случайного порядка выступлений на стендап-встречах с хранением данных в Vercel KV.

## 🚀 Особенности

- ✅ **Современный дизайн** с неоморфическим стилем и темной темой
- ✅ **Облачное хранилище** через Vercel KV (Redis)
- ✅ **Работает в инкогнито** - без localStorage
- ✅ **Синхронизация между устройствами** - все видят изменения
- ✅ **Быстро и надежно** - 30,000 запросов/месяц бесплатно
- ✅ **Адаптивный дизайн** для всех устройств

## 🏗️ Архитектура

```
Frontend → Vercel API → Vercel KV (Redis)
```

1. **Frontend** - пользовательский интерфейс
2. **Vercel API** - серверные функции для работы с данными
3. **Vercel KV** - облачное хранилище данных (Redis)

## 📋 Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/your-username/standup-randomizer.git
cd standup-randomizer
```

### 2. Задеплойте на Vercel

1. Перейдите на [vercel.com](https://vercel.com) и создайте аккаунт
2. Нажмите **Import Project**
3. Выберите этот репозиторий
4. Нажмите **Deploy**

### 3. Настройте Vercel KV

**Следуйте подробной инструкции: [VERCEL_KV_SETUP.md](VERCEL_KV_SETUP.md)**

Краткая версия:
1. В проекте Vercel → **Storage** → **Create Database**
2. Выберите **KV (Redis)**
3. Создайте базу данных
4. Нажмите **Connect to Project** и выберите ваш проект
5. Передеплойте проект

### 4. Обновите конфигурацию (опционально)

В файле `public/script.js` замените URL на ваш Vercel домен:

```javascript
const API_CONFIG = {
    baseUrl: 'https://your-project.vercel.app', // Ваш Vercel URL
    endpoint: '/api/update-participants'
};
```

## 🔧 Преимущества Vercel KV

### Почему Vercel KV?

- ✅ **Никаких токенов** - автоматическая настройка
- ✅ **30,000 команд/месяц** бесплатно (обновляются!)
- ✅ **Быстро** - Redis, глобальная репликация
- ✅ **Работает везде** - даже в режиме инкогнито
- ✅ **Простая интеграция** - один клик

### Мониторинг использования

1. Vercel проект → **Storage** → Ваша KV база
2. Вкладка **Usage** показывает статистику

## 🐛 Устранение неполадок

### Ошибка "Server configuration error"
- Убедитесь, что Vercel KV база создана и подключена к проекту
- Проверьте, что проект передеплоен после подключения базы

### Список не загружается
- Откройте консоль браузера (F12) и проверьте ошибки
- Проверьте, что `baseUrl` в `script.js` соответствует вашему Vercel домену
- Убедитесь, что база данных активна в Vercel → Storage

### CORS ошибки
- Проверьте настройки CORS в API файлах
- Убедитесь, что запросы идут на правильный домен
